# Laravel URL Shortener

Веб-приложение на Laravel 12 для создания коротких ссылок, отслеживания переходов, сбора статистики кликов и управления ссылками через пользовательскую панель и административную панель Filament.

## 1. Обзор проекта

Laravel URL Shortener — это веб-приложение для создания и управления короткими ссылками через пользовательскую панель. Каждая созданная короткая ссылка перенаправляет посетителя на исходный URL и автоматически сохраняет информацию о переходе, включая IP-адрес и время посещения.

Приложение разделяет пользовательскую и административную части. Зарегистрированные пользователи могут создавать, просматривать, отслеживать и удалять только свои ссылки, а администратор может управлять данными приложения через административную панель Filament v3, построенную на существующих моделях Eloquent.

Проект был разработан для демонстрации практических навыков backend-разработки на Laravel, включая аутентификацию и авторизацию пользователей, проектирование базы данных, работу с Eloquent ORM, валидацию запросов, построение связей между моделями и реализацию CRUD-операций.

## 2. Возможности проекта

### Аутентификация

- Регистрация и вход пользователей с использованием Laravel Breeze.
- Защита маршрутов с помощью middleware аутентификации Laravel.
- Каждый пользователь имеет доступ только к своим ссылкам и личному кабинету.

### Сокращение ссылок

- Создание коротких ссылок из исходных URL.
- Генерация уникальных 6-символьных short code.
- Проверка на уникальность short code перед сохранением в базе данных.

### Управление ссылками

- Просмотр всех ссылок, принадлежащих авторизованному пользователю.
- Удаление существующих ссылок.
- Автоматическое удаление истории переходов при удалении ссылки (Cascade Delete).

### Статистика переходов

- Автоматическое перенаправление посетителей на исходный URL.
- Автоматическая запись каждого перехода.
- Сохранение IP-адреса посетителя.
- Сохранение даты и времени каждого перехода.
- Отображение общего количества переходов для каждой ссылки.
- Просмотр полной истории переходов, отсортированной от новых к старым.

### Безопасность

- Серверная валидация URL.
- Пользовательские сообщения об ошибках валидации.
- Проверка авторизации для предотвращения просмотра и удаления чужих ссылок.

### Административная панель Filament

- Управление ссылками через Filament v3.
- Управление историей переходов через Filament v3.
- Поиск записей.
- Сортировка записей.
- Отображение связей Eloquent между моделями.

## 4. Структура базы данных

Приложение использует три основные таблицы базы данных. Каждая таблица отвечает за свою задачу и связана с другими таблицами с помощью внешних ключей, что обеспечивает целостность данных.

### Структура базы данных

```text
+----------------------+
|        users         |
+----------------------+
| id (PK)              |
| name                 |
| email                |
| password             |
+----------------------+
           │
           │ links table uses user_id (FK) to reference users.id in users table.
           │
           ▼
+----------------------+
|        links         |
+----------------------+
| id (PK)              |
| user_id (FK)         |
| original_url         |
| short_code           |
+----------------------+
           │
           │ clicks table uses link_id (FK) to reference link.id in users table.
           │
           ▼
+----------------------+
|       clicks         |
+----------------------+
| id (PK)              |
| link_id (FK)         |
| ip_address           |
| created_at           |
+----------------------+
```

### Связи между таблицами

```text
Один пользователь
    │
    └────── может создать много ссылок

Одна ссылка
    │
    ├────── принадлежит одному пользователю
    │
    └────── может иметь много переходов

Один переход
    │
    └────── относится к одной ссылке
```

### Назначение таблиц

**users**

Хранит зарегистрированных пользователей, которые могут входить в систему и управлять своими короткими ссылками.

**links**

Хранит все короткие ссылки, созданные авторизованными пользователями. Каждая ссылка принадлежит одному пользователю и содержит исходный URL и сгенерированный короткий код.

**clicks**

Хранит информацию о каждом переходе по короткой ссылке. При каждом открытии короткой ссылки создаётся новая запись, благодаря чему приложение может хранить историю переходов, IP-адреса посетителей, время переходов и подсчитывать общее количество кликов.

## 5. Архитектура приложения

Приложение построено по архитектуре MVC (Model–View–Controller), используемой в Laravel. Каждый компонент отвечает только за свою задачу, благодаря чему код остаётся понятным, поддерживаемым и легко расширяемым.

### Общая архитектура

```text
Браузер
    │
    ▼
Маршруты (web.php)
    │
    ▼
Контроллеры
    │
    ▼
Модели (Eloquent ORM)
    │
    ▼
База данных MySQL
    │
    ▼
Blade-представления
    │
    ▼
Ответ браузеру
```

### Назначение компонентов

#### Маршруты (Routes)

Маршруты принимают HTTP-запросы от браузера и определяют, какой метод контроллера должен их обработать.

Примеры:

- `GET /dashboard`
- `POST /links`
- `GET /{shortCode}`
- `GET /links/{link}/stats`
- `DELETE /links/{link}`

---

#### Контроллеры (Controllers)

Контроллеры содержат основную логику приложения.

Их задачи:

- Проверка входящих данных.
- Генерация уникальных коротких ссылок.
- Получение и сохранение данных в базе данных.
- Запись статистики переходов.
- Перенаправление пользователей.
- Передача данных в Blade-представления.

---

#### Модели (Models)

Модели Eloquent представляют таблицы базы данных и определяют связи между ними.

```text
User
    │
    └── hasMany(Link)

Link
    ├── belongsTo(User)
    └── hasMany(Click)

Click
    └── belongsTo(Link)
```

Эти связи позволяют Laravel автоматически получать связанные записи без написания SQL JOIN-запросов вручную.

---

#### Blade-представления (Blade Views)

Blade-шаблоны отвечают за отображение данных, полученных от контроллеров.

Примеры:

- Dashboard
- Страница статистики
- Страницы аутентификации

Blade отвечает только за отображение данных и не содержит основной логики приложения.

---

#### База данных

База данных MySQL хранит все данные приложения.

- `users` — зарегистрированные пользователи.
- `links` — сокращённые ссылки.
- `clicks` — история переходов по ссылкам.

Доступ к базе данных осуществляется через Eloquent ORM.

## 6. Поток работы приложения

Приложение обрабатывает каждый запрос по архитектуре MVC (Model–View–Controller). Любой запрос пользователя проходит через определённую последовательность этапов, после чего Laravel возвращает готовый ответ в браузер.

### Жизненный цикл запроса

```text
Браузер
    │
    ▼
Маршрут (Route)
    │
    ▼
Контроллер
    │
    ▼
Модель (Eloquent ORM)
    │
    ▼
База данных MySQL
    │
    ▼
Контроллер
    │
    ▼
Blade-представление / Перенаправление
    │
    ▼
Ответ браузеру
```

### Объяснение потока

1. Браузер отправляет HTTP-запрос.

2. Laravel находит подходящий маршрут (Route), определённый в файле `routes/web.php`.

3. Найденный маршрут вызывает соответствующий метод контроллера.

4. Контроллер выполняет необходимую логику приложения, например:
    - проверяет входящие данные;
    - получает данные из базы данных;
    - создаёт уникальный короткий код;
    - записывает информацию о переходах;
    - перенаправляет пользователя.

5. Контроллер обращается к моделям Eloquent для получения или изменения данных в базе данных.

6. База данных возвращает найденные записи, а Eloquent преобразует их в объекты моделей и передаёт обратно контроллеру.

7. После выполнения всех действий контроллер:
    - передаёт данные в Blade-представление для отображения, **или**
    - выполняет перенаправление пользователя на другую страницу.

8. Laravel формирует итоговый HTTP-ответ и отправляет его обратно в браузер.

## 7. Поток аутентификации

В этом разделе показано, что происходит при входе пользователя в систему.

### Поток входа в систему

```text
Браузер

↓

POST /login

↓

Маршрут (Route)

POST /login

↓

Контроллер аутентификации Laravel

↓

Проверка email и пароля

↓

Модель User

↓

Таблица users

↓

Данные верны?

├── Нет
│     ↓
│  Вернуть ошибки валидации
│
└── Да
      ↓
Создать пользовательскую сессию

↓

Перенаправить на:

GET /dashboard

↓

Маршрут (Route)

↓

LinkController::index()

↓

Модель Link

↓

Таблица links

↓

dashboard.blade.php

↓

Браузер
```

### Объяснение потока

1. Пользователь отправляет форму входа.

2. Laravel сопоставляет запрос с маршрутом:

```text
POST /login
```

3. Контроллер аутентификации Laravel проверяет введённые данные.

4. Модель `User` выполняет поиск пользователя по адресу электронной почты в таблице `users`.

5. Если данные введены правильно, Laravel создаёт пользовательскую сессию.

6. Laravel перенаправляет пользователя на:

```text
GET /dashboard
```

7. Маршрут `/dashboard` вызывает метод:

```php
LinkController::index()
```

8. Метод `LinkController::index()` получает все ссылки, принадлежащие авторизованному пользователю.

9. Полученные объекты модели `Link` передаются в:

```text
dashboard.blade.php
```

10. Laravel формирует страницу Dashboard и возвращает её в браузер.

## 8. Поток создания короткой ссылки

Этот процесс начинается, когда авторизованный пользователь вводит URL в форме на Dashboard для создания короткой ссылки.

### Поток создания короткой ссылки

```text
Авторизованный пользователь

↓

Dashboard

↓

POST /links

↓

Маршрут (Route)

↓

LinkController::store(Request $request)

↓

Проверка введённого URL

↓

Генерация уникального short code

↓

Модель Link

↓

Таблица links

↓

Создание новой записи

↓

Перенаправление на:

GET /dashboard

↓

Маршрут (Route)

↓

LinkController::index()

↓

Модель Link

↓

Получение всех ссылок авторизованного пользователя

↓

dashboard.blade.php

↓

Браузер
```

### Объяснение потока

1. Авторизованный пользователь вводит URL в форму на Dashboard и нажимает кнопку **Create Short Link**.

2. Браузер отправляет запрос:

```text
POST /links
```

3. Laravel сопоставляет запрос с маршрутом:

```php
Route::post('/links', [LinkController::class, 'store']);
```

4. Laravel вызывает метод:

```php
LinkController::store(Request $request)
```

5. Контроллер проверяет введённый пользователем URL.

6. Приложение генерирует уникальный шестисимвольный `short_code`.

7. Модель `Link` создаёт новую запись в таблице `links`, содержащую:

- ID авторизованного пользователя;
- исходный URL;
- сгенерированный `short_code`.

8. После успешного создания ссылки Laravel выполняет перенаправление на:

```text
GET /dashboard
```

9. Laravel сопоставляет запрос с маршрутом:

```php
Route::get('/dashboard', [LinkController::class, 'index']);
```

10. Laravel вызывает метод:

```php
LinkController::index()
```

11. Модель `Link` получает все ссылки, принадлежащие авторизованному пользователю.

12. Полученные объекты модели `Link` передаются в:

```text
dashboard.blade.php
```

13. Blade формирует обновлённую страницу Dashboard, на которой отображается только что созданная короткая ссылка.

## 9. Поток перенаправления и записи перехода

Этот процесс начинается, когда посетитель открывает короткую ссылку.

### Поток перенаправления и записи перехода

```text
Посетитель открывает:

GET /eR7aM8

↓

Маршрут (Route):

GET /{shortCode}

↓

Laravel присваивает:

{shortCode} = eR7aM8

↓

LinkController::redirect($shortCode)

↓

Модель Link

↓

Поиск в таблице `links`, где:

links.short_code = eR7aM8

↓

Найденная запись сохраняется в объект модели:

$link

↓

Модель Click

↓

Создание новой записи в таблице `clicks`:

clicks.link_id = $link->id
clicks.ip_address = IP-адрес посетителя
clicks.created_at = текущее время

↓

Перенаправление посетителя на:

$link->original_url
```

### Объяснение потока

1. Посетитель открывает короткую ссылку, например:

```text
http://127.0.0.1:8000/eR7aM8
```

2. Laravel сопоставляет запрос с маршрутом:

```php
Route::get('/{shortCode}', [LinkController::class, 'redirect']);
```

3. Laravel извлекает значение после символа `/` и сохраняет его в параметр маршрута:

```php
$shortCode = "eR7aM8";
```

4. Laravel вызывает метод:

```php
LinkController::redirect($shortCode)
```

5. Модель `Link` выполняет поиск в таблице `links`, где:

```text
links.short_code = eR7aM8
```

6. Если запись найдена, Eloquent преобразует её в объект модели `Link` и сохраняет в переменную:

```php
$link
```

7. Модель `Click` создаёт новую запись в таблице `clicks`, содержащую:

```text
link_id     = $link->id
ip_address  = request()->ip()
created_at  = текущее время
```

8. Laravel перенаправляет посетителя по адресу, хранящемуся в:

```php
$link->original_url
```

9. Браузер автоматически открывает исходный URL.

## 10. Поток просмотра статистики

Этот процесс начинается, когда авторизованный пользователь открывает страницу статистики для одной из своих ссылок.

### Поток просмотра статистики

```text
Авторизованный пользователь

↓

Нажимает:

View Statistics

↓

GET /links/8/stats

↓

Маршрут (Route):

GET /links/{link}/stats

↓

Laravel присваивает:

{link} = 8

↓

Laravel использует Route Model Binding

↓

Модель Link выполняет поиск:

links.id = 8

↓

Найденная запись сохраняется в объект модели:

$link

↓

LinkController::stats(Link $link)

↓

Проверка авторизации:

$link->user_id = auth()->id()

↓

Использование связи `clicks()`

↓

Поиск в таблице `clicks`, где:

clicks.link_id = $link->id

↓

Сортировка записей от новых к старым

↓

Полученные объекты модели Click сохраняются в:

$clicks

↓

Передача `$link` и `$clicks` в:

links/stats.blade.php

↓

Blade отображает:

Исходный URL
IP-адреса посетителей
Время переходов

↓

Браузер
```

### Объяснение потока

1. Авторизованный пользователь нажимает кнопку **View Statistics** для выбранной ссылки.

2. Blade формирует URL с помощью:

```php
route('links.stats', $link->id)
```

Например:

```text
links.id = 8
```

преобразуется в:

```text
GET /links/8/stats
```

3. Laravel сопоставляет запрос с маршрутом:

```php
Route::get('/links/{link}/stats', [LinkController::class, 'stats'])
    ->name('links.stats');
```

4. Laravel присваивает:

```text
{link} = 8
```

5. Так как метод контроллера объявлен как:

```php
LinkController::stats(Link $link)
```

Laravel автоматически использует модель `Link` для поиска записи, где:

```text
links.id = 8
```

6. Eloquent преобразует найденную запись базы данных в объект модели `Link` и сохраняет его в:

```php
$link
```

7. Контроллер проверяет, принадлежит ли ссылка текущему авторизованному пользователю:

```php
abort_if($link->user_id !== auth()->id(), 403);
```

8. Затем контроллер вызывает:

```php
$link->clicks()->latest()->get();
```

9. Связь `clicks()` выполняет поиск в таблице `clicks`, где:

```text
clicks.link_id = $link->id
```

10. Метод `latest()` сортирует найденные записи по полю `created_at` в порядке от новых к старым.

11. Метод `get()` получает найденные записи в виде коллекции объектов модели `Click` и сохраняет её в:

```php
$clicks
```

12. Контроллер передаёт обе переменные в представление:

```php
return view('links.stats', [
    'link' => $link,
    'clicks' => $clicks,
]);
```

13. Представление `links/stats.blade.php` отображает:

```php
$link->original_url
$click->ip_address
$click->created_at
```

14. Laravel формирует страницу статистики и отправляет её обратно в браузер.

## 11. Административная панель Filament

Проект включает административную панель Filament v3, которая предоставляет готовый интерфейс для управления данными приложения.

Вместо ручной разработки контроллеров, Blade-представлений, таблиц, форм, поиска, сортировки и пагинации Filament автоматически создаёт административный интерфейс на основе существующих моделей Eloquent.

Административная панель Filament использует ту же базу данных, те же модели и те же связи между моделями, что и основное приложение.

### Архитектура административной панели

```text
Браузер

↓

/admin

↓

Filament

↓

Resource

↓

Модель Eloquent

↓

База данных MySQL

↓

Resource

↓

Интерфейс Filament

↓

Браузер
```

### Ресурс Link

`LinkResource` связан с моделью `Link`.

```php
protected static ?string $model = Link::class;
```

Это сообщает Filament, что данный ресурс использует модель `Link`, которая представляет таблицу `links`.

Ресурс Link отображает:

- ID ссылки
- Владельца ссылки
- Исходный URL
- Короткий код (Short Code)
- Общее количество переходов
- Дату создания

Также ресурс предоставляет возможность:

- Поиска
- Сортировки
- Редактирования
- Удаления

### Ресурс Click

`ClickResource` связан с моделью `Click`.

```php
protected static ?string $model = Click::class;
```

Это сообщает Filament, что данный ресурс использует модель `Click`, которая представляет таблицу `clicks`.

Ресурс Click отображает:

- ID перехода
- Короткий код связанной ссылки
- ID ссылки
- IP-адрес посетителя
- Время перехода

Ресурс также демонстрирует использование связей Eloquent с помощью:

```php
Tables\Columns\TextColumn::make('link.short_code')
```

Поток получения данных:

```text
ClickResource

↓

Модель Click

↓

clicks.link_id

↓

links.id

↓

links.short_code

↓

Отобразить Short Code
```

Административная панель Filament была добавлена как бонусная часть проекта, чтобы показать, как существующее Laravel-приложение можно расширить полноценной административной панелью, повторно используя существующие модели Eloquent, связи между моделями и данные приложения.
